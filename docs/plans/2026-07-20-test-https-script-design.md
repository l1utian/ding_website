# 测试证书 HTTPS 脚本设计

## 目标

新增独立 Bash 脚本 `scripts/enable-test-https.sh`，使用 `certs/tech.gd.cn.crt` 和 `certs/tech.gd.cn.key` 为两个生产域名启用 HTTPS，并把 HTTP 请求永久跳转到 HTTPS。

## 边界

- 不修改 `scripts/deploy-production.sh`。
- 后续普通生产部署仍选择“非首次部署”，只替换静态文件，不触碰 Nginx。
- 腾讯云安全组的 TCP 443 放行由用户在控制台完成，脚本不调用腾讯云 API。
- 证书为测试证书，浏览器出现不受信任提示属于预期结果。
- 不启用 HSTS，避免测试证书影响后续浏览器访问。

## 执行流程

1. 检查本地证书、私钥以及 `ssh`、`scp` 命令。
2. 将证书、私钥和远程执行脚本上传到服务器临时目录。
3. 在服务器检查 Nginx、OpenSSL、站点目录和现有生产配置。
4. 校验证书有效期、域名范围以及证书与私钥是否匹配。
5. 备份现有 Nginx 配置，安装证书并设置私钥权限为 `600`。
6. 写入 80 端口跳转和两个 443 站点配置。
7. 执行 `nginx -t`；失败时恢复备份并返回失败状态。
8. reload Nginx，并用 `curl -k` 验证两个 HTTPS 站点。

## 安全与失败处理

- 使用 `set -Eeuo pipefail`，任何未处理错误立即暴露。
- 所有远程路径固定，不接受任意目标路径输入。
- 修改配置前创建带时间戳的备份。
- 配置检查或 reload 失败时恢复备份，再明确返回失败。
- 私钥只上传到临时路径和 `/etc/nginx/ssl/tech.gd.cn`，权限为 `600`。
- 将 `/certs/` 加入 `.gitignore`，避免私钥进入 Git。

## 验证

- 自动测试检查脚本存在、Bash 语法正确、包含备份/校验/跳转/HTTPS/回滚逻辑，并确认未修改生产部署脚本。
- 运行完整 Vitest 和 TypeScript 类型检查。
- 不在本地验证阶段实际连接或修改生产服务器；用户执行脚本时才发生远程变更。
